6. 访问控制权限

  • default (即缺省，什么也不写）: 在同一包内可见，不使用任何修饰符。使用对象：类、接口、变量、方法。

  • private : 在同一类内可见。使用对象：变量、方法。 注意：不能修饰类（外部类）

  • public : 对所有类可见。使用对象：类、接口、变量、方法

  • protected : 对同一包内的类和所有子类可见。使用对象：变量、方法。 注意：不能修饰类（外部类）。

2. Error ,Exception 与 RuntimeException 的区别与联系

  • 继承结构：Error 和 Exception 都是继承于 Throwable，RuntimeException 继承自 Exception。
  • Error 指的是 JVM 错误，如内存空间不足，栈溢出等，编译器不检查。
  • Exception 表示程序可以处理的异常，分为 Runtime Exception（编译器不检查，如除以0） 和 Checked Exception（编译器检查，如文件找不到）
  • 常见的 RuntimeException： ArithmeticException、 NullPointerException、 ClassCastException

4. throws 与 throws 的区别

二十四、 Annotation 注解

java SE 支持自定义注解的开发，并且提供了三个常用的注解： @Override @Deprecated @SuppressWarnings

1. @Override ——为了准确的重写

eg. 重写时写错了父类方法名，编译器会认为这是新方法，不予报错
解决： 在方法上一行加上 @Override 告诉编译器下面的方法为重写的方法，编译器就会进行检查。

2. @Deprecated ——为了准确的重写

eg. 之前的方法不再使用时, 在方法上一行加上 @Deprecated 告诉编译器下面的方法不再使用，此时方法名上会出现一道横线。

3. @SuppressWarnings ——压制警告

eg. 代码中出现黄线，欲去除，可以用压制警告方法，不需自己写，直接在 IDE 中操作即可生成。

3. 方法引用 （为方法设置别名）（了解）

  • 引用静态方法： 类名称 :: static 方法名称
  • 引用某个对象的方法： 实例化对象 :: 普通方法名称
  • 引用特定类型方法： 特定类 :: 普通方法名称
  • 引用构造方法： 类名称 :: new

方法的引用需要有一个函数式接口，该接口要有注解 FunctionInterface 即接口中只能定义一个方法, 函数需设置好参数

Java 1.8 定义了四种函数式接口：

需要 import java.util.function.接口名

  • 功能型（Function）： 有参数，有返回值
    public interface Function<T,R>{public R apply(T, t);}
  • 消费型（Consumer）： 有参数，没有返回值
    public interface Consumer{public void accept(T, t);}
  • 供给型（Supplier）： 没有参数，有返回值
    public interface Supplier{public R get();}
  • 断言型（Predicate）： 进行判断
    public interface Predicate{public R test(T, t);}

二十六、 Java 线程

1 继承 Thread 类, 重写父类 run（） 方法, 通过 子类对象.start() 方法启动

使用 Thread 类的 start() 方法不仅仅是启动多线程的执行代码，还要根据不同的操作系统进行资源的分配

2. 实现 Runnable 接口，实现run（） 方法，创建 Thread 类的对象并将线程类对象作为参数传进去，通过线程类对象.start() 启动

3. 多线程两种实现方式的区别

  • 使用 Runnable 接口解决了 Java 的单继承问题
  • Thread 类实现了 Runnable 接口， 通过 MyThread 类实现 Runnable 接口后，再通过 Thread 类来启动线程，看起来更像 代理模式 ，只是代理模式客户端使用的方法应该为接口中提供的方法（Run方法）
  • Runnable 接口可以比 Thread 类能够更好地描述数据共享（即多个线程访问统一资源的操作）的概念

4. 第三种实现方式——Callable接口 可以让线程返回结果(from JDK1.5)

5. 线程的命名与取得

打印流

产生原因：

字节流 Outputstream 输出时，只接受 byte 类型，即使是 FileWriter 类，也只是扩展参数到int，String，char[]。如果想输出其他类型比如 double， boolean等需要转换，十分麻烦，此时可以使用打印流类

  • 字节打印流： PrintStream
  • 字符打印流： PrintWriter

使用：
以PrintStream 为例，它是 OutputStream 的间接子类，而创建其对象时需要接受一个 OutputStream 的对象，使用的是 装饰设计模式， 将父类的功能进行拓展，并且可以接收指向不同子类对象的父类引用

  • 字符串： %s
  • 小数： %m.nf

控制台输入输出

总结：

  • InputStream 和 OutPutStream 是最高层的 字节流 抽象类, 常常以子类 FileInputStream 和 FileOutputStream 进行实例化
  • Reader 和 Writer 是最高层的 字符流 抽象类，常常以子类 FileReader 和 FileWriter 进行实例化
  • 字节流和字符流之间的转换使用 转换流 InputStreamReader 和 OutPutStreamWriter 来进行
  • 使用 System.in/System.out 将输入/输出定向到 控制台/屏幕
  • 使用 内存流 ByteArrayInputStream/ByteArrayOutputStream 将输入/输出 定向到内存
  • 对于普通字节/字符流输出类型单一的改进： 使用 打印流 PrintStream/PrintWriter

  • 对于普通字节/字符流输入类型单一的改进： 使用 缓冲流 BufferedReader,可以支持String， 再升级后使用 Scanner

  • 以后输出使用打印流 PrintStream/PrintWriter， 输入使用 Scanner
