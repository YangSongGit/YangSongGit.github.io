五、 方法的定义与使用

1.方法的重写与重载

  • 重写(Override)：又称“覆盖”。子类继承了父类的方法并重新进行实现，返回值，函数名与参数列表都不变且访问权限不能比父类中被重写的方法的访问权限更低

  • 重载(Overload)：同一个类中，方法名相同，参数列表不同，返回值类型可不同(开发时一般让其相同)

六、 面向对象

1. 面向对象三大特点：

  • 封装性： 保护内部结构的安全性
  • 继承性： 在原有代码上扩充新的功能
  • 多态性： 同一个行为具有多个不同表现形式

2. 堆内存与栈内存

  • 堆内存： 保存对象的属性内容，需要用关键字new来开辟
  • 栈内存： 保存的是一块堆内存的地址

七、 类与对象

1. 构造方法

  • 开发建议：类之中可以有多个构造方法，但是必须保留一个无参构造方法（已定义其他构造方法则类将不会提供默认无参构造方法）

  • 构造方法不一定全为 public ，私有构造器的使用场景：

    如果想要在返回对象引用之前做一些额外的工作或者想记录到底创建了多少个对象（可能要限制其数量），用私有构造函数加上静态的成员函数的方法是大有裨益的 —— Java 编程思想

    私有构造方法+静态成员函数调用 可以用来实现单例模式和工厂方法模式，具体见Java 私有构造函数的应用

2. 类的加载过程

  • 准备阶段：先为静态变量分配内存并设置为默认值（注意是默认值，比如int类型为0，这时尚未执行java代码；特殊情况：如果为final static则初始化为指定的值，因为final定义常量，在编译的时候就分配了）
  • 类初始化阶段（执行类构造器()方法的过程,()方法是由编译器自动收集类中的所有类变量的赋值动作和静态语句块static{}中的语句合并产生的）：为静态变量赋值，执行静态语句块（此时静态语句块可以访问并赋值定义在前面的静态变量，对定义在后面的静态变量只能赋值不能访问。

static 声明的属性可以由类名直接访问，在对象没有初始化时也可访问，因为类的初始化过程要先初始化static属性

写在方法中的代码块，其中变量为局部变量，作用域即为该代码块

十三、 继承

1. 子类对于父类的私有属性的继承属于隐式继承，不能直接访问，对于其他的属性属于显式继承。

2. 实例化子类对象之前会先调用父类的无参构造

父类中没有无参构造方法时，子类的构造方法中需要用：
super(参数列表)
显示的调用父类的有参构造，否则报错。

3. super() 与 this() 不能同时出现

4. 子类 B 中方法 fun() 重写(即覆盖)了父类 A 方法 fun() ，则 B.fun() 会调用子类方法，想要调用父类方法使用 super.fun()

注意：

  • 重写时子类中的方法不能有比父类中该方法更严格的访问权限
  • 如果父类中方法 fun() 为 private， 而子类中也有该方法且为 public，虽然满足上一规则，但是这不是重写，因为父类中 private 方法对于子类不可见，子类相当于自己定义了一个 fun() 方法。

5. final 关键字

  • final 定义的类不能被继承
  • final 定义的方法不能被子类重写
  • final 定义的变量是常量，常量名用全部大写表示。

十四、 多态性

1. 方法的多态性

  • 重载（overloading）：根据传入参数的类型及个数来调用不同的方法
  • 重写（override）：根据子类的不同来调用不同的操作，实现不同的功能

2. 对象的多态性

  • 向上转型（自动完成）： 父类 父类对象 = 子类实例 //即父类引用指向子类对象
  • 向下转型（强制完成）： 子类 子类对象 = （子类） 父类实例

向上转型可以通过父类引用指向不同的子类对象（子类已经重写父类方法），在执行某种操作时根据不同的子类类型来实现不同的功能

3. 抽象类不能用 final 定义， 因为 final 定义的类不能被继承
